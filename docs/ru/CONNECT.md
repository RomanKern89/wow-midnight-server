# Подключение клиента (RU)

Как направить ретейл-клиент **12.0.7.68275** на твой сервер.

> [English](../en/CONNECT.md)

## 1. Совпадение билда

Билд клиента **обязан** совпадать с билдом базы мира сервера (**68275**).
Используй инструмент вроде лаунчера **Arctium** / **BNetInstaller** для установки
и поддержания нужного ретейл-билда. Сами файлы клиента — собственность Blizzard,
получай их только через свою легальную установку.

## 2. Патч клиента под кастомный сервер

Ретейл-клиенты подключаются через Battle.net с проверкой сертификата/портала.
Стандартный подход для приватных серверов — **Arctium Game Launcher**, который
патчит запущенный клиент в памяти, чтобы он принимал кастомный **портал** и
логин-сертификат сервера. Файлы игры при этом не изменяются навсегда.

Типичный порядок:
1. Положи лаунчер Arctium рядом с `WoW.exe` (ретейл-билд 68275).
2. Настрой **портал** на хост твоего сервера (адрес, который анонсирует
   `bnetserver`).
3. Запускай через Arctium, а не через лаунчер Blizzard.

## 3. Логин-сертификат (dev cert)

`bnetserver` предъявляет логин-сертификат. Для приватной установки это **dev-
сертификат** TrinityCore, отдаваемый по REST-эндпоинту логина (порт 8081).
Убедись, что:
- `bnetserver.conf` `LoginREST.ExternalAddress` / `LoginREST.Port` доступны
  клиенту.
- `address` реалма в `auth.realmlist` — это IP, до которого клиент достучится.

> Логин современного ретейла идёт через **HTTP REST dev-cert**, а не через
> старый realmlist.wtf. Если логин «крутится» — почти всегда дело в адресе/порте
> REST-эндпоинта или в несовпадении билда. Проверяй это в первую очередь.

## 4. Первый вход

1. Запуск через Arctium → экран логина Battle.net.
2. Введи аккаунт, созданный через `account create`.
3. Выбери реалм → ты в игре.

## 5. Диагностика

| Симптом | Вероятная причина |
|---|---|
| Зависло на «Connecting» / «Authenticating» | Недоступен REST-эндпоинт логина (8081) или неверный портал |
| «Version mismatch» / не пускает на логине | Билд клиента ≠ билд сервера 68275 |
| Пустой список реалмов | Неверный `auth.realmlist.address` или worldserver (8085) выключен |
| Персонаж создан, но нет квестов | Пробелы в данных/цепочках квестов — см. [FIXES.md](FIXES.md) |

Спутники-боты: в игре `.bot` (боты-существа) и `.pbot` (боты-игроки) — см.
[FEATURES.md](FEATURES.md).
